# 01 - 系统总览

## 设计目标

本系统旨在解决三个核心问题：

1. **信息碎片化**：每天接触大量信息，但分散在浏览器、微信、邮件、视频平台
2. **关联缺失**：笔记越记越多，但彼此孤立，无法形成知识网络
3. **复用困难**：需要时想不起来，或者想起来但找不到

## 五层架构

```
┌─────────────────────────────────────────────┐
│                 输入层                        │
│   文本 / 链接 / 视频 / 语音 / PDF / 想法      │
└──────────────────┬────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│               处理层                          │
│        Claude Code / LLM 处理               │
│   采集 → 清洗 → 转录 → 结构化 → 打标签       │
└──────────────────┬────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│               关联层                          │
│        NotebookLM 跨源语义关联                │
│   跨源检索 → 语义关联 → 发现隐藏联系         │
└──────────────────┬────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│               沉淀层                          │
│           Obsidian 模板化沉淀                 │
│   笔记模板 → frontmatter → 双链 → 索引       │
└──────────────────┬────────────────────────────┘
                   │
                   ▼
┌─────────────────────────────────────────────┐
│               复盘层                          │
│        索引 / 日报 / 项目上下文               │
│   自动维护 → 周期复盘 → 持续迭代               │
└─────────────────────────────────────────────┘
```

## 核心流程

一次完整的信息处理闭环：

```
用户输入
   ↓
Claude Code 采集并结构化
   ↓
NotebookLM 查询历史知识关联
   ↓
生成带关联发现的结构化笔记
   ↓
写入 Obsidian 对应目录
   ↓
更新索引 / context / 日报
   ↓
等待下一次输入
```

## 关键收益

- **从被动收藏到主动加工**：每个输入都经过 LLM 处理和你的思考
- **从孤立笔记到知识网络**：NotebookLM 和 Obsidian 双链共同构建关联
- **从手动整理到自动维护**：脚本承担重复劳动，你只管输入和思考
- **从短期记忆到长期资产**：沉淀后的笔记可以被搜索、引用、复用

## 与其他系统的区别

| 方案 | 特点 | 本系统 |
|---|---|---|
| 纯 Obsidian | 手动记录 | LLM 辅助采集与关联 |
| 纯 Notion | 数据库驱动 | Markdown + 双链驱动 |
| 单一 ChatGPT | 对话式 | 结构化沉淀 + 自动化维护 |
| 传统 PKM | 重分类 | 轻分类、重关联、重复用 |
