# 02 - 输入层

## 目标

统一接收所有潜在有价值的输入，避免信息散落在微信、浏览器、邮件、视频平台、备忘录等各个角落。

## 输入类型

| 类型 | 示例 | 采集方式 |
|---|---|---|
| 文本 | 一段想法、聊天记录、会议记录 | 直接粘贴 |
| 链接 | 文章、视频、播客、论文 | 网页抓取 / 转录 |
| 视频 | YouTube、B站、课程 | 字幕提取 |
| 语音 | 录音、播客 | 语音转文字 |
| PDF | 论文、报告、电子书 | 文本提取 |
| 图片 | 截图、白板 | OCR + 描述 |
| 灵感 | 突发想法 | 快速记录到 inbox |

## 输入原则

1. ** inbox 是缓冲区，不是终点**
   - inbox 只负责快速捕获
   - 定期清空 inbox，进入处理层

2. **不要过度分类输入阶段**
   - 先存下来，再处理
   - 不要在输入时纠结放到哪个目录

3. **保留原文**
   - 原始素材必须完整保存
   - 处理后的笔记不能替代原文

## 输入层目录设计

```text
vault/
├── inbox/          # 临时捕获，定期清空
└── raw/            # 原始素材存档
```
