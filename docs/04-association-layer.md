# 04 - 关联层

## 目标

发现新输入与已有知识之间的关联，避免笔记孤立。

## 为什么需要关联层？

- 人脑擅长联想，但不擅长记住所有细节
- 随着笔记数量增加，跨笔记关联变得越来越难
- 仅靠 Obsidian 双链需要主动建立链接，容易遗漏

## NotebookLM 的作用

NotebookLM 在本系统中承担**跨源语义关联**的角色：

1. 将历史资料作为 source 上传
2. 对当前输入进行查询
3. 返回与当前主题相关的历史 source
4. 提取相关原文摘要
5. 将关联发现写入新笔记

## 关联流程

```
结构化后的新输入
   ↓
NotebookLM 查询历史知识库
   ↓
返回相关 source 列表
   ↓
逐个拉取 source 原文
   ↓
提取核心段落
   ↓
写入「关联发现」区块
```

## 关联发现格式

```markdown
## 关联发现

- **源标题 1**：相关原文摘要...
- **源标题 2**：相关原文摘要...
```

## 设计原则

1. **不能只列引用编号**
   - 必须附上源标题和原文摘要
   - 否则无法判断关联价值

2. **原文摘要控制在 300-500 字**
   - 太长会淹没新笔记
   - 太短无法体现关联

3. **关联是发现，不是结论**
   - AI 提供关联线索
   - 是否采纳由人判断

## 下一步

关联完成后，笔记进入 [05 - 沉淀层](05-sedimentation-layer.md)，写入 Obsidian 并统一归档。
