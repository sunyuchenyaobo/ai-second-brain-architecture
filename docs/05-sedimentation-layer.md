# 05 - 沉淀层

## 目标

将处理后的信息以统一的格式、清晰的结构、可检索的方式长期保存。

## 核心工具：Obsidian

Obsidian 在本系统中承担**长期知识库**的角色：

- **本地优先**：数据完全属于你自己
- **Markdown 原生**：格式通用、可迁移
- **双链支持**：笔记之间可以互相引用
- **插件生态**：Templater、Dataview 等插件可大幅提升效率

## 目录结构

```text
vault/
├── inbox/          # 临时捕获，定期清空
├── raw/            # 原始素材存档
├── research/       # 研究笔记
├── projects/       # 项目笔记
├── daily-notes/    # 日记/日报
├── people/         # 人物笔记
├── templates/      # 笔记模板
└── context.md      # 流水线上下文
```

## 沉淀原则

1. **每个笔记都有 frontmatter**
   - 标准化元数据，便于检索和自动化处理

2. **统一笔记结构**
   - 相同类型的笔记使用相同的区块结构

3. **原文 → 核心观点 → 我的思考 → 关联发现**
   - 保留原文
   - 提炼核心观点
   - 加入个人思考
   - 补充关联发现

4. **用双链连接相关笔记**
   - `[[笔记名]]` 语法
   - 构建知识网络

## 笔记类型

| 类型 | 用途 | 目录 |
|---|---|---|
| research | 研究学习 | `research/` |
| project | 项目记录 | `projects/` |
| daily | 日记/日报 | `daily-notes/` |
| person | 人物笔记 | `people/` |
| idea | 灵感 | `inbox/` 或 `research/` |
