# 06 - 复盘层

## 目标

通过索引、日报、项目上下文等方式，让系统保持有序、可追踪、可迭代。

## 复盘内容

### 1. 索引更新

每新增一篇研究笔记，自动更新 `research/00-Research索引.md`：

- 按日期分组
- 新条目插入对应日期区块最上方
- 便于按时间回顾

### 2. context.md

记录当前流水线的上下文：

- 本次做了什么
- 结论是什么
- 下次应该继续什么

下次启动时先读取，无缝衔接。

### 3. 日报/周报

定期生成：

- 今日输入
- 今日沉淀
- 关键发现
- 明日待办

### 4. 项目复盘

每个项目笔记定期更新：

- 当前进展
- 阻塞点
- 下一步动作
- 关键决策

## 复盘原则

1. **自动化重复劳动**
   - 索引更新交给脚本
   - 日报可半自动生成

2. **人工确认关键结论**
   - AI 辅助整理
   - 人来做最终判断

3. **复盘不是形式**
   - 目标是为了迭代和复用
   - 不要为了复盘而复盘

## 下一步

复盘层的很多工作可以通过 [07 - 自动化层](07-automation-layer.md) 的脚本来自动完成。
