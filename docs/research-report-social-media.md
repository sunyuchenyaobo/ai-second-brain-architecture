## 二、爆款共性分析

### 2.1 标题公式

高流量标题基本符合以下几个结构：

**公式 1：身份 + 工具 + 效果**
- "我如何用 obsidian+AI 做知识管理+内容创作？"
- "I Turned Claude Into the Ultimate Second Brain"

**公式 2：极端词 + 教程**
- "手把手带你从0搭建"
- "全网千万收藏"
- "保姆级教学"
- "一学就会"

**公式 3：名人/权威 + 同款**
- "复现 Karpathy 的 LLM Wiki 同款知识库"
- "搭建卡帕西同款知识库"
- "Andrej Karpathy posted a simple idea..."

**公式 4：反常识/争议**
- "obsidian才是最适合ADHD的第二大脑"
- "AI is giving me brain rot"
- "我奶都会用obsidian"

### 2.2 封面/首图特征

虽然没有拿到具体封面数据，但从标题可以推断：

- **前后对比型**：「Before/After 大脑状态」
- **工具截图型**：Obsidian 界面 + 箭头标注
- **数字清单型**："3 分钟"、"5 步"、"30 天"
- **冲突型**："别再...让AI来"

### 2.3 内容结构共性

**短视频/图文爆款结构**：

1. **钩子（前 3 秒）**：用一个痛点抓人
   - "你是不是收藏了 1000 篇，但从来想不起来看？"
   - "你的大脑是不是快炸了？"

2. **展示结果（前 15 秒）**：展示系统/效果
   - "这是我用 AI 搭的第二大脑"
   - "它再也没失忆过"

3. **简化步骤（中间）**：3-5 个步骤
   - 输入 → 处理 → 关联 → 沉淀
   - 或者：Claude → NotebookLM → Obsidian

4. **收尾 CTA**：
   - "代码开源，评论区自取"
   - "关注我看后续"

### 2.4 情绪共鸣点

| 痛点 | 爆款表达 | 触发人群 |
|---|---|---|
| 信息焦虑 | "收藏了 1000 篇但想不起来" | 知识打工人 |
| 记忆焦虑 | "它再也没失忆过" | 担心遗忘的人 |
| 效率焦虑 | "别再手动整理了，让AI来" | 工具控 |
| 学习焦虑 | "我奶都会用obsidian" | 新手 |
| AI 替代恐惧 | "AI is giving me brain rot" | 开发者 |
| 工具选择困难 | "Claude + Obsidian = UNSTOPPABLE" | AI 工具爱好者 |

### 2.5 平台差异

| 平台 | 用户偏好 | 爆款特征 |
|---|---|---|
| 小红书 | 图文、短平快、情绪化 | 标题带 emoji、口语化、痛点直接 |
| B站 | 教程、实操、长视频 | "保姆级"、"手把手"、"从零" |

| Reddit | 经验分享、讨论 | 真实实验、反常识、长文 |

## 三、可复用的流量设计技巧

### 3.1 选题层面

1. **蹭名人/权威**
   - Karpathy、Andrej Karpathy 的 "second brain" 概念在 Twitter 上爆了
   - 国内可以蹭 "卡帕西同款"、"Karpathy 推荐"

2. **制造对比/反差**
   - "我奶都会用obsidian" — 降低门槛
   - "AI is giving me brain rot" — 反向焦虑
   - "收藏 1000 篇 vs 真正会用" — 痛点对比

3. **数字+承诺**
   - "3 分钟学会"
   - "30 天实验"
   - "5 步搭建"
   - "10 个 agent"

4. **身份标签**
   - "21 岁大学生"
   - "PhD student"
   - "AI 产品经理"
   - 身份让内容更可信赖

### 3.2 标题层面

**高点击率标题模板**：

1. "我用 [工具 A] + [工具 B] 做了 [效果]，[夸张副词]"
   - 例：我用 Claude + NotebookLM + Obsidian 搭了套系统，再也没失忆过

2. "[数字] 分钟/步/天，[动作]"
   - 例：3 分钟搭建 AI 第二大脑

3. "[权威/名人] 同款 [效果]，[动作]"
   - 例：复现 Karpathy 同款 AI 知识库

4. "别再 [旧方式]，让 [AI/工具] 来 [新方式]"
   - 例：别再手动整理笔记了，让 AI 来

5. "[反常识观点]：为什么 [工具] 是 [人群] 的最佳选择"
   - 例：为什么 Obsidian 才是 ADHD 患者的最强第二大脑

### 3.3 内容层面

1. **开头 3 秒必须抓人**
   - 用痛点提问："你是不是也..."
   - 用反常识结论："很多人没意识到..."
   - 用结果前置："这是我用 AI 搭的第二大脑"

2. **中间给地图，不给细节**
   - 展示系统架构图
   - 说清 3-5 个步骤
   - 不要讲太细，引导到 GitHub/公众号

3. **结尾留钩子**
   - "代码开源，评论区自取"
   - "下期讲自动化脚本"
   - "关注我，带你从零搭建"

### 3.4 视觉层面

1. **架构图必备**
   - 五层架构图是核心视觉资产
   - 用 Excalidraw 或 Mermaid 画出"高级感"

2. **界面录屏**
   - Obsidian 双链图谱
   - Claude Code 操作界面
   - NotebookLM source 关联

3. **前后对比**
   - "以前我的笔记 vs 现在我的笔记"
   - "没有 AI vs 有 AI"

### 4.3 首发内容建议

**首发平台**：B站（适合教程）+ 小红书（适合引流）

**首发标题**（B站）：
> 《21 岁大学生用 Claude+NotebookLM+Obsidian 搭了套 AI 第二大脑｜代码开源》

**首发标题**（小红书）：
> 《21 岁，我把 AI 接上了第二大脑，再也没失忆过》

**首发脚本结构**：

```
【0-3 秒】钩子
"你是不是也收藏了 1000 篇，但从来想不起来看？"

【3-10 秒】结果展示
"这是我用 Claude+NotebookLM+Obsidian 搭的第二大脑。"
"它能把看过的文章自动关联起来，再也不会失忆。"

【10-40 秒】核心架构
"整个系统分 5 层：
输入 → Claude 处理 → NotebookLM 关联 → Obsidian 沉淀 → 自动复盘。
输入一篇文章，Claude 抓重点，
NotebookLM 找到你三个月前写过的相关笔记，
Obsidian 把这一切都沉淀成结构化的研究笔记。"

【40-50 秒】开源
"我把整个工作流开源到 GitHub 了，
clone 下来就能当 Obsidian vault 用。"

【50-60 秒】CTA
"评论区放链接。
下期讲自动索引脚本怎么写。"
```

### 4.4 内容节奏建议

**第一周**：发 1 条爆款钩子视频 + 1 条架构解析
**第二周**：发 1 条实操教程 + 1 条小红书图文
**第三周**：发 1 条 "30 天实验" 复盘 + 1 条避坑指南
**第四周**：发 1 条 Q&A + 1 条下期预告

### 4.5 引流路径设计

```
公域内容
  ↓
评论区 / 主页 → GitHub 仓库
  ↓
仓库 README 引导关注公众号
  ↓
公众号沉淀系列教程
  ↓
建立社群 / 付费产品
```

### 4.6 需要避免的坑

1. **不要过度承诺**
   - 避免"用了就能记住所有知识"
   - 强调：AI 辅助，人做判断

2. **不要展示私人笔记**
   - 所有截图用 demo 数据

3. **不要一次性讲太细**
   - 短视频 3-5 个点
   - 长视频 10 分钟内
   - 引导到 GitHub 或公众号

4. **不要只发一次**
   - 一个选题多平台分发
   - 一个选题拆成系列
